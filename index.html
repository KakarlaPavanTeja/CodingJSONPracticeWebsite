Reference & Guides

How to Use

  1. Choose Question Type: Select "Standard" or "Node-Based" (affects C++ repository structure).
  2. Choose Action: Select "Create New", "Update Existing", or the dedicated ID Changer.
  3. Upload Files: Upload the .lua config file and the .json test cases file for Create/Update, or just the coding_questions.json for ID Changer.
  4. Generate & Download: The tool generates the final coding_questions.json.

Tool Features & Output Expectations

  • Create & Update: Supports both generating new question JSONs and modifying existing ones while preserving key IDs.
  • ID Changer Mode: This mode takes an existing JSON file and regenerates all question, code, solution, and test case IDs, treating it as a new question copy. It also ensures the time limits are output in the correct float format (e.g., `1.0`).
  • Question Level Validation: The tool only accepts question levels of EASY, MEDIUM, or HARD (case-insensitive).
  • Conditional ID Preservation (Update Mode):
    • If the number of new test cases is equal to the number of existing test cases, the UUIDs of the test cases, the Solutions ID, the Question ID, and the Code Content IDs are preserved.
    • If the number of new test cases is increased or decreased, all UUIDs associated with the question (test cases, solutions, question, and code content) are replaced with new IDs to ensure correct ordering and data integrity.
  • Detailed Error Reporting: The UI displays the exact error message from any exceptions encountered during processing.
  • Metadata Fields: The final JSON metadata includes real_life_example, follow_up_questions (with content_type: MARKDOWN for answers), and a comprehensive topics array combining all difficulty tags.

Hints Configuration Details

The tool generates hint titles (e.g., "Hint 1") and sets the unlock time sequentially:

  • Hint 1 unlocks at 10 seconds.
  • Hint 2 unlocks at 20 seconds.
  • Hint N unlocks at N * 10 seconds.

Provide the description content (Markdown is supported) for each hint in the .lua file:

----------HINTS_START----------

----------HINTS_START_1----------
Description for the first hint.
----------HINTS_END_1----------

----------HINTS_START_2----------
Description for the second hint.
----------HINTS_END_2----------

----------HINTS_END----------

Solution Code Configuration Details

The tool only includes a language's solution in the final JSON if content is present between its START/END tags. If all solution sections are empty, "solutions": [] will be output.

----------SOLUTIONS_CPP_START----------
// Add C++ solution code here (optional)
----------SOLUTIONS_CPP_END----------

----------SOLUTIONS_PYTHON_START----------
# Add Python solution code here (optional)
----------SOLUTIONS_PYTHON_END----------

----------SOLUTIONS_JAVA_START----------
// Add Java solution code here (optional)
----------SOLUTIONS_JAVA_END----------

Test Case Expectations (.json)

  • Validation: Every test case must include input, weightage, and order.
  • Multi-Output Validation: If multiple_possible_output: true is set, the outputs array must also be present and populated. If not, the standard output field is expected.
  • Total Score Calculation: The final total_score field in the output JSON is derived by summing the weightage of all input test cases.
  • Hidden Test Cases: Test cases at index 0 and 1 are set to is_hidden: false. All subsequent test cases (index 2 onwards) are set to is_hidden: true.
