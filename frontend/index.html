Input contract reference

Use these checked-in examples and rules to prepare compatible files without depending on external templates.

Workflow

  1. Choose Create or Update.
  2. Select Standard or Node-Based and execution style.
  3. Select only languages present in the Lua file.
  4. Upload Lua, testcases, and existing JSON when updating.
  5. Resolve blocking validation errors before downloading.
  6. Share the downloaded JSON, Lua, testcase source, and summary with the internal team for platform loading.

Testcase requirements

  • Non-empty string input.
  • String output, or non-empty outputs with multiple_possible_output: true.
  • Strictly positive numeric weightage.
  • Orders are normalized sequentially.
  • At least one valid tag per testcase.
  • Tags become platform name/display objects.

Checked-in examples

Output guarantees

  • Exactly one default enabled language.
  • UTF-8-safe repository encoding.
  • Positive weights and exact total score.
  • Stable IDs during updates, including count changes.
  • Unknown top-level legacy fields remain intact.
